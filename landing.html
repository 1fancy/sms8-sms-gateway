How it works

Three steps to ship SMS from your AI tool

Setup is the same whether you use Claude Code, Cursor, Windsurf, Codex, or any other MCP-compatible AI assistant.

STEP 01

Get an API key

Sign up at app.sms8.io. Pair your Android phone via QR. Copy your API key from Profile then API.

STEP 02

Add the MCP server

Paste a 6-line config into your AI tool, or run the one-command plugin install for Claude Code.

STEP 03

Ask in plain English

"Add SMS verification to my signup flow." The AI calls send_otp and verify_otp and writes the code.

Seven tools

What your AI assistant can do

Every tool re-uses the same send pipeline as the SMS8 dashboard. Credits, retries, multi-device routing, and webhook signing behave identically to a direct API call.

setup_sms8

One-shot handshake. Validates the API key, returns devices, plan and integration context to the AI.

send_sms

Send a single SMS through a paired Android. Supports per-device and per-SIM routing.

send_otp

Generate and send a verification code. Configurable length, expiry and attempt limits.

verify_otp

Constant-time compare against the latest OTP for that phone. Returns remaining attempts on mismatch.

get_messages

Fetch recent inbox or sent messages. Filter by direction or phone for reply flows and debugging.

list_devices

List paired Android devices. Pick the sender device when load-balancing across multiple SIMs.

create_webhook

Register a callback URL for inbound SMS and delivery events. HTTPS only, SSRF-protected, HMAC-signed.

Three install paths

Pick whichever fits your tool

SMS8 ships as a hosted HTTP MCP server, a Claude Code plugin with a built-in Skill, and an npx launcher for stdio clients. All three use the same backend.

  • Hosted HTTP works in Claude Code, Cursor and Windsurf with a Bearer token
  • Claude plugin via /plugin marketplace add 1fancy/sms8-sms-gateway
  • npx launches the stdio bridge: npx -y @sms8/mcp

Production-ready

Security defaults, not foot-guns

Hard per-phone cap

5 OTPs per number per 24 hours. Not user-configurable. Protects recipients even if your API key leaks.

Race-proof rate limits

Cooldown and cap checks wrapped in DB transactions with row locks. Parallel callers cannot race past the limit.

POST-only OTP endpoints

GET returns 405. Cookies ignored. Eliminates CSRF via image tags and accidental browser preloads.

API key redaction

setup_sms8 returns only the last 4 chars of your key so it never leaks into AI chat history.

SSRF-guarded webhooks

Blocks loopback, RFC1918, CGNAT, link-local, IPv4-mapped IPv6, and resolves the host at validation time.

HMAC-signed deliveries

Inbound SMS and delivery events are signed with HMAC-SHA256. Constant-time compare on the receiving end.

FAQ

Frequently asked questions

What is the SMS8 MCP server?

An HTTPS endpoint at mcp.sms8.io that implements the Model Context Protocol so AI coding tools (Claude Code, Cursor, Windsurf, Codex, Devin) can send SMS, issue one-time passwords, and configure webhooks. SMS routes through a paired Android phone via the SMS8 platform.

How do I add SMS to a Claude Code project?

Add the MCP server to ~/.config/claude/mcp-servers.json with the HTTP transport pointing at https://mcp.sms8.io and your SMS8 API key as a Bearer token. Or run /plugin marketplace add 1fancy/sms8-sms-gateway then /plugin install sms8-sms-gateway. After restarting Claude Code, ask: add SMS notifications to this app via the sms8 MCP.

Does this work with Cursor and Windsurf?

Yes. Both support HTTP MCP servers. Add https://mcp.sms8.io with your SMS8 API key as a Bearer header to ~/.cursor/mcp.json (Cursor) or ~/.codeium/windsurf/mcp_config.json (Windsurf). The setup wizard at app.sms8.io/mcp-setup.php prefills the JSON per tool.

Can the AI add OTP verification automatically?

Yes. The MCP exposes send_otp and verify_otp. Defaults: 6-digit code, 5-minute expiry, 5 attempts, 60-second resend cooldown. The bundled Claude Code Skill teaches the assistant when to invoke these tools and how to wrap an existing signup or login route.

Is this a Twilio alternative?

Yes. SMS8 uses your own Android phone as the gateway with your existing SIM card. No per-message fees, no A2P 10DLC registration, no phone-number provisioning. Flat pricing from $16/month and unlimited SMS.

Is the source code public?

Yes, MIT-licensed. The server, all 7 tools, the npm launcher, the Claude Code plugin manifest, and production-ready code examples are at github.com/1fancy/sms8-sms-gateway.

Is it safe to use in production?

Yes. Server-side defences include per-key rate limiting, a hard cap of 5 OTPs per phone per 24h, transaction-wrapped check + insert (race-proof), HMAC-SHA256 webhook signatures, comprehensive SSRF block list on webhook URLs, POST-only OTP endpoints, and API-key redaction in setup_sms8 responses.

Can I self-host the MCP server?

Yes. Clone the repo, set SMS8_APP_PATH to your SMS8 install directory, and run php -S 127.0.0.1:8080 index.php. The hosted server at mcp.sms8.io is free and the easier path for most users.
